# CSS Styling Example

This page demonstrates how to style callouts using **plain CSS** with no JavaScript
components. [`@r4ai/remark-callout`](https://github.com/r4ai/remark-callout) is
configured to emit custom HTML elements (`callout-root`, `callout-title`, `callout-body`),
which can then be styled entirely in CSS.

Unlike the [Custom Components](/custom-components) example, this approach requires
**zero JavaScript** for rendering â€” making it ideal for purely static sites.

## Callouts with Rich Content

Callouts can contain any Markdown content â€” lists, code, headings, and more.

> [!example] Configuration
> Add `remark-callout` to your `astro.config.ts`:
>
> ```ts
> import { remarkCallout } from "@r4ai/remark-callout"
> import { defineConfig } from "astro/config"
>
> export default defineConfig({
>   markdown: {
>     remarkPlugins: [remarkCallout],
>   },
> })
> ```

> [!note] Supported callout types
> The following types are recognized out of the box:
>
> - `note`, `info`, `todo`
> - `abstract`, `summary`, `tldr`
> - `tip`, `hint`, `important`
> - `success`, `check`, `done`
> - `question`, `help`, `faq`
> - `warning`, `caution`, `attention`
> - `failure`, `fail`, `missing`
> - `danger`, `error`
> - `bug`
> - `example`
> - `quote`, `cite`

## Foldable Callouts

Add `-` after the type to make a callout collapsible (collapsed by default),
or `+` to make it collapsible but open by default.

> [!note]- Collapsed by default
> This callout starts **collapsed**. Click the title to expand it.
>
> Hidden content is revealed on expand â€” great for long notes or spoilers.

> [!tip]+ Open by default (foldable)
> This callout starts **open** but can be collapsed.
>
> Useful for optional information that most readers may want to see.

> [!warning]- Read before proceeding
> Make sure you have a full backup before running the following command.
>
> ```sh

## How It Works

This page uses the default HTML output from `remark-callout`, which emits
custom elements (`callout-root`, `callout-title`, `callout-body`) styled via
`src/styles/callout.css`.

**No React components are used** â€” just CSS attribute selectors and custom
properties (CSS variables) for per-type theming.

The key pattern in `callout.css`:

```css
/* Base container */
callout-root {
  display: block;
  border-radius: 0.5rem;
  border: 1px solid rgba(var(--callout-color) / 0.25);
  background-color: rgba(var(--callout-color) / 0.08);
  padding: 1rem;
  margin-block: 1.5rem;
}

/* Per-type colour via CSS custom property */
callout-root[type="note"]    { --callout-color: 59 130 246; }  /* blue-500  */
callout-root[type="success"] { --callout-color: 34 197 94;  }  /* green-500 */
callout-root[type="warning"] { --callout-color: 249 115 22; }  /* orange-500 */
callout-root[type="danger"]  { --callout-color: 239 68 68;  }  /* red-500   */
```

> [!tip] Foldable callouts and CSS
> Native foldable behaviour (`<details>`/`<summary>`) works out of the box when
> you configure `remark-callout` to preserve the default HTML elements for
> foldable callouts. When using custom element names (as in this example),
> foldable callouts fall back to plain block elements â€” consider using the
> [Custom Components](/custom-components) approach if you need interactive
> folding without JavaScript workarounds.
